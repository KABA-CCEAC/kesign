Choose Application Type

This section will help you to decide for the right application type and appropriate interface style.

If you got any uncertainties feel free to contact the Kaba User Interface Design Officer for guideance.

Webapplikation

Doing a responsive webapplikation is in most cases the best and cost efficient solution. If the applikation is developed with the mobile first approach for interaction there is not even the need for a separate solution for the mobile.

You might consider not doing a webapplikation if you have special requirements:

  • to device sensors
  • of webapplikation would be distracting
  • only once for all devices
  • cost than native apps
  • is possible
  • access to device sensors
choose the interface style of kesign

Native Application

If an webapplikation is not sufficent. You should consider doing an application to the devices native style.

You might consider not doing a native application if you have special requirements:

  • would be improved by custom application
  • to the specific devices
  • cost than webapplikation
  • access to device sensors
  • branding
  • development environment
  • via store
choose the style of the given device (iOS, Android, Win8,...)

Custom Application

If an native application is not sufficent. You should consider doing an application to the devices native style.

If ever possible try to reuse an existing kaba design style (eg. b-inspired).

You might consider choosing a custom application if:

  • a kaba device with display (try reuse of b-inspired style)
  • usability wins (eg. user is used to terminal usage, reuse of terminal style might be better than native style experience)
  • to the specific devices
  • cost than native applikation
  • access to device sensors
  • style will raise costs
  • development environment
  • via store
reuse a kaba design style or create a special (b-inspired)
